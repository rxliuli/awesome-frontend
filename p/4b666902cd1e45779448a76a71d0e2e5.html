# 框架及社区

# react (opens new window): 前端流行的 mvc 框架

对比 react vue
优点 生态庞大 门槛很低，开发者友好
IDE 支持完善 文档是前端框架中最好的
灵活
与 TypeScript 结合完美
缺点 官方基本放任社区发展 IDE 支持一直都不那么完美
对开发者体验不太上心，这会在下面的社区中反复体现到 与 ts 结合不好，vue3 也未能解决模板的类型问题

# 脚手架和打包工具

# vite: 一个非常优秀的前端脚手架

对比 vite create-react-app snowpack
优点 非常快 react 官方脚手架 react 是一等公民
支持 ts 配置 非常快
开发者友好
缺点 vue 是一等公民 强制锁定版本 存在许多兼容性问题

若非 snowpack 实在不争气，吾辈也不至于把 vite 作为 react 脚手架的首选

# rollup (opens new window): 前端 npm 库打包工具

对比 rollup webpack esbuild
优点 易于使用 非常灵活 非常快
库友好 生态庞大 支持 js/go API 和插件
缺点 生态还不是那么丰富 灵活是一把双刃剑 使用 go 编写，生态上可能存在问题

# 工程化

# prettier (opens new window): 代码统一格式化工具

# eslint (opens new window): 前端可配置的 linter 工具（实测还是没有 WebStorm 的内置 linter 好用）

# env-cmd (opens new window): 跨平台设置环境变量

# husky (opens new window): 为项目配置 git 钩子

推荐使用 v4

# lint-staged (opens new window): 针对暂存的 git 文件运行 linters

# patch-package (opens new window): 在本地覆盖任何有问题的 npm 包中的文件

主要是处理一些 npm 模块可能存在小问题但又来不及提 PR 的情况下，在本地修改并生成 git patch 文件，在每次 yarn install 时合并这些 patch。

  • 修复包不支持 cjs 模块，参考：https://github.com/remarkjs/remark-math/issues/56 (opens new window)
  • 修复 typescript 类型定义错误，参考：https://stackoverflow.com/questions/67701504 (opens new window)

# ts-jest (opens new window): 支持 ts 单元测试的预设配置

# vuepress: 静态文档网站生成器

很多对比已经在 VuePress 官网上的 为什么不是 (opens new window) 里面说明了，吾辈只对比 vuepress 与 docusaurus。

对比 vuepress docusaurus
优点 开发者友好 基于 react
功能非常齐全
缺点 配置还是多了点 翻译基于配置而非 markdown
修改 js 配置无法热更新 开发者体验并不是那么友好

和 snowpack 一样，基于 react 的解决方案实在不争气。
